1. Що таке Github?
Відповідь: GitHub - це сайт для спільної роботи з різноманітними проектами та кодами.
2. Що таке репозиторій?
Відповідь: Репозиторій – це плейс, де розробники можуть слідкувати та керувати своїми проектами, кодом та файлами. Також власник репозиторію може надавати доступ користувачам, які можуть допомогти з проектом.
3. Що таке гілка?
Відповідь: Гілки дозволяють розробникам носити та тестувати зміни в ізольованому середовищі, не торкаючись основного коду.
4. Навіщо потрібен GitHub Desktop?
Відповідь: GitHub Desktop – це графічний інтерфейс для спрощення роботи з репозиторіями на GitHub. Він забезпечує зручний спосіб взаємодії GitHub без необхідності використання командного рядка.
5. Що таке commit?
Відповідь: Commit - це дія, при якій раніше внесені зміни в коді чи файлах фіксуються та зберігаються в історії репозиторію. Це дозволяє відстежувати, хто, коли та які зміни вніс до проекту.
6. Що таке pull request?
Відповідь: Pull request - це запит від розробника власнику репозиторію з урахуванням його змін в основну гілку проекту.
7. Що таке merge request?
Відповідь: Merge request - це запит на злиття також є запитом від розробника до власника репозиторію, із зазначенням змін, внесених в окрему гілку, в основному гілку проекту.
